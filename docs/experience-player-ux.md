## 1. 目的

訪れた人が **5秒で「番号を押せばいい」** と分かり、ガイドのあと **現場の言葉で聞いて社内ボットを想像できる** こと。

旧 Live/AI シェル（パック切替・左サイド二重ナビ・空チャット待ち）は公開導線に使わない。

---

## 2. ルート分担

| ルート | 役割 | 誰向け |
|--------|------|--------|
| `/manufacturing` | ハブ（楔＋3カード＋組み合わせ文） | 公開 |
| `/play/:packId` | 体験プレイヤー（本命UI） | 公開 |
| `/?pack=…` | 互換。公開リンクは `/play/…` へ誘導 | 移行中 |
| `/?pack=…&packs=1` または旧 Live | 探索シェル（パック切替あり） | 開発・営業 |
| `/ai?pack=…` | 当面はプレイヤー内のライブに統合。単独は互換 | 移行中 |

公開の正: **ハブ → `/play/minato-factory`（①）**

## 3. 画面構成（1画面の仕事順）

```text
[ヘッダー] テーマ名 · ハブへ戻る
[1] ガイドステップ（主操作）
[2] 回答＋根拠（主結果）
[3] 現場の言葉で聞く（本命・ガイド進行後に前面）
[4] 次の一歩（折りたたみ）ライブ / 自社文書 / ROI / 相談 / 他テーマ
```

### 3.1 常に見せる

1. **テーマ1行**（例: 製造① 現場判断 — 文書に聞いてその場で動く）
2. **ガイド番号**（①は5手。押すと回答）
3. **最新の回答＋出典**

### 3.2 ガイド1手以上（またはスキップ）のあと前面

4. **現場の言葉で聞く**
   - 見出し: 「現場の言い方のまま聞ける。根拠がある範囲だけ答えます」
   - チップ（塗装剥がれ等）＋自由入力
   - 社内ボット連想を明示

### 3.3 折りたたみ「次の一歩」（ガイド完了または明示オープン後）

5. ライブ接続（サンプル／APIキー／体験コード）— Access Mode 部品を再利用
6. 自社文書を足す — 「自社ルールで試せます」を主文言に
7. ROI・相談・他テーマ（②③）・ハブ

### 3.4 出さない（公開 play）

- 現場バー（全パック切替）
- 左サイドの文書／質問二重ナビ
- 体験中の ROI 常時オーバーレイ
- ガイド前からの自由入力常時表示
- Starter 等のテンプレパック

## 4. 体験フロー（感情の設計）

| 段 | ユーザーの行動 | 感じてほしいこと |
|----|----------------|------------------|
| A ガイド | 番号を押す | 判断の型が分かる |
| B 現場言葉 | 自由記述またはチップ | 社内ボットになりそう |
| C ライブ＋自社資料 | 接続・アップロード・再質問 | うちのルールでも使える |
| D 次へ | ROI／相談／他テーマ | 導入の話に進める |

**本命は B。** A は入口、C は本格化、D は出口。

---

## 5. 旧シェルとの関係

- 機能追加の継ぎ足し先は旧シェルにしない
- `sampleEngine` / `aiEngine` / Access Mode / Trial / ROI / Contact はプレイヤーが呼ぶ
- `?packs=1` で旧 Live を開ける（開発用）
