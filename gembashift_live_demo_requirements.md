# ConformSystem Live Demo 要件定義書

## 0. 文書概要

| 項目 | 内容 |
|---|---|
| **目的** | 説明文なしで、仕様書について聞いた瞬間に根拠付きの答えが返る体験をさせる |
| **位置づけ** | デプロイして URL を渡す／同席で開く **稼働中の製品体験** |
| **対になる文書** | [ConformSystem_demo_web_requirements.md](./ConformSystem_demo_web_requirements.md)（LP・理解用） |
| **基本思想** | 売らない。説明しない。触らせる。中心は SaaS 管理画面ではなく **質問 → 正確な回答 → 根拠確認** |

### LP との役割分離

| | LP | Live Demo（本要件） |
|---|---|---|
| 役割 | 理解させる | 体験させる |
| 順序 | 読む → 理解 → 納得 → 試す | 触る → 驚く → 理解する |
| 初画面 | Hero・45分・動画・説明 | **操作画面のみ** |
| 入口 | `/lp` | `/`（デプロイ主入口） |
| 維持 | 現行を維持 | 本要件で新規 |

中心の一本道:

```text
URLを開く
 → すでに仕様書が読み込まれている
 → 質問する
 → 回答が返る
 → 根拠をクリック
 → 該当条項とページが開く
```

---

## 1. 成功条件

1. 「LPだ」ではなく「ツールが開いた」と感じる
2. スクロールせず、すでに1回答が見える
3. 送信で正確な回答（数値・差分・影響・条項）が返る
4. 根拠を押すと条項・ページが開く
5. クリックした UI はすべて意味のある反応をする

失敗条件:

- 初画面にキャッチコピー・課題統計・CTA・Before/After 説明
- 常時3カラムの管理画面感が質問体験を邪魔する
- 動かないセレクタなど偽の操作 UI
- 4つの大きな質問ボタンだけが動くデモ感
- 巨大な DEMO バナー

---

## 2. 画面仕様

### 2.1 既定レイアウト（PC）— 2カラム

出典は常設しない。質問体験に集中する。

```text
┌────────────────────────────────────────────────────────────┐
│ ConformSystem   温度制御ユニット改訂 Sample   v3.2 → v3.4  Ready │
├──────────────┬─────────────────────────────────────────────┤
│ DOCUMENTS    │  （初期）差分Q&A 表示済み                     │
│ ● 制御仕様書 │  [ 3件の根拠を確認 → ]                       │
│   v3.4       │─────────────────────────────────────────────│
│ ○ 制御仕様書 │  この変更の影響範囲は？              送信    │
│   v3.2       │                                             │
└──────────────┴─────────────────────────────────────────────┘
```

「根拠を確認」クリック時のみ右ペインを開く:

```text
Docs | 質問・回答 | 出典（§ / p. / 抜粋ハイライト）
```

### 2.2 モバイル

- メインは質問・回答
- サイドバーはドロワー
- 出典はボトムシート

### 2.3 Top bar

| 要素 | 内容 |
|---|---|
| 製品名 | ConformSystem |
| 案件 | 温度制御ユニット改訂 + 小さく `Sample` |
| 版 | **ラベル** `v3.2 → v3.4`（偽セレクタ禁止） |
| 状態 | Ready |

巨大バナーや「これはサンプルです！」は置かない。`Demo workspace` または `Sample` 程度。

### 2.4 原則: 偽操作 UI を作らない

- 操作できそうに見えるものは、本当に操作できる
- 動かせないならラベルにする
- 文書一覧の v3.2 / v3.4 クリックは **アクティブ文書切替＋出典フィルタ** として実働させる

### 2.5 初期状態（空画面禁止）

- スレッドにシナリオ1（温度センサー差分）の Q&A を表示済み
- 入力欄にシナリオ2「この変更の影響範囲は？」をプリセット
- 出典ペインは閉じた状態
- 文書は v3.4 をアクティブ

### 2.6 質問マッチング（4ボタン感を消す）

- 大きな4チップ列を主役にしない
- 複数言い回しを同一シナリオへマッピング
- 未マッチは生成せず、上品な空結果＋近いサジェスト（入力セット用）

### 2.7 出典

- オンデマンド表示
- 文書名・版・ページ・条項・抜粋・ハイライト
- アクティブ文書版の抜粋を優先

## 7. 最終方針

```text
聞いたら、正確な答えが返る。
根拠を押したら、仕様書の該当箇所が開く。
```

それだけでよい。SaaS の箱を作ることが目的ではない。
